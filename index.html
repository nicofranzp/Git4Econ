QUÉ ES GIT?

QUÉ ES GIT?

  • Git es un sistema distribuido de control de versión. El más utilizado en el mundo!
  • El software de Git se instala "localmente". Para empezar a utilizar Git no es necesario un servidor. No obstante, este último mejora la colaboración significativamente
  • Git no es lo mismo que GitHub! GitHub es una red social que utiliza la arquitectura Git como base. Otros ejemplos son GitLab y BitBucket

CÓMO SE USA GIT?

CÓMO SE USA GIT?

  • Git se usa a través de la linea de comando (shell, console, terminal, etc.)

  • Git es parte de la distribución de otros softwares, por lo que puede que ya esté instalado. (e.g., Python-Anaconda, o command line tools de MacOs, Matlab 2020b)

CÓMO SE USA GIT?

CÓMO SE APRENDE GIT?

  • La mejor forma de aprender Git es usándolo!!
  • La mayor parte de la documentación es a través de la línea de comando.
  • Esto no quiere decir que el usuario lo use a través de ella, pero le da uniformidad al proceso (Las GUI pueden tener formas diversas de implementar Git y aprenderlas es difícil si no se entiende el objetivo. Es como querer aprender MS excel sin saber matemáticas)
  • Con el tiempo, el uso y la terminología Git se vuelve natural

  • Git en la línea de comando puede tomar algún tiempo.
  • Las interfaces de usuario facilitan el proceso, pero involucran preferencias personales.
  • Una de las UI populares es VS Code de Microsoft
    • Integración Git desde su instalación
    • Variados lenguages de programación y herramientas que facilitan la escritura de código.
    • Marketplace con extensiones gratuitas
    Desde VS code se puede utilizar y compilar LaTeX y markdown. Se puede utilizar Matlab, python y R. Se puede tener un sistema de punteros TODO y FIXME. Sistemas de corrección ortográfico (tanto para texto como código), sistema de brackets colorizados, etc

Colaboración con Git

  • Anteriormente mencionamos que git no requiere servidores.
  • En teoría, podríamos compartir un repositorio utilizando un disco duro en común
  • También se pueden crear permisos de lectura y escritura (ver documentación de git init
  • No obstante, un servidor git permite crear un espacio común y seguro!
  • Git ocupa los siguientes protocolos de comunicación:
    • Local
    • HTTP y HTTPS
    • SSH
    • GIT

Colaboración con Git

  • Para colaborar con git sólo nos hace falta aprender los siguentes comandos
    • git remote
    • git clone
    • git pull (git fetch + git commit)
    • git push

Sin embargo hay un componente humano que hace falta

Redes sociales

  • Tener la información no es lo mismo que entender la información
  • Este espacio lo llenan las redes sociales, que permiten dar fluidez a la comunicación
    • Archivos Readme y Wiki
    • Issues
    • Forks
    • Pull requests
  • Las mas famosas son GitLab, GitHub y BitBucket.
  • Se pueden intalar en redes privadas.

Comentarios finales

  • Git es un secretario que lleva registro completo del repositorio.
  • La mejor forma de aprenderlo es usándolo.
  • Git no es mágico. Requiere que los equipos definan estrategias para crear ramas y resolver conflictos.
  • Su utilidad aumenta exponencialmente cuando se siguen los siguentes tips:
    • Hacer commits frecuentes y con sentido
    • Usar ramas para experimentar y combinarlas cuando los experimentos funcionan
    • Actualizar el repositorio constantemente
  • Las GUI's ayudan a humanizar git
  • Las redes sociales en torno a git son herraminetas de colaboración y comunicación muy poderosas. Digitalizan espacios análogos y evitan muchas reuniones y videollamadas
