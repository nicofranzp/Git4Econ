Solución *: Coordinando una forma de hacer y registrar los cambios

"naive"

  • Si no registramos cambios de forma sistemática, nuestra capacidad de identificar cambios esta sujeta a la "maldición de la dimensionalidad"
    • Riesgos de continuidad operacional (innovar es costoso)
    • Poca colaboración y replicabilidad (sentido común?)
    • La historia se pierde (generalmente en la carpeta "old")

"naive"

  • Los programadores han tenido este problema por décadas.
  • Las soluciones (software) han ido evolucionando con el tiempo. En general se les conoce como sistemas de control de versión
  • La mayoría de ellos presenta al usuario la versión que se quiere revisar y deja el resto fuera de la vista.

  • La mayoría de ellos presenta al usuario la versión que se quiere revisar y deja el resto fuera de la vista.
  • Permiten identificar cambios entre versiones porque registran la primera derivada.

QUÉ ES GIT?

QUÉ ES GIT?

  • Git es un sistema distribuido de control de versión. El más utilizado en el mundo!
  • El software de Git se instala "localmente". Para empezar a utilizar Git no es necesario un servidor. No obstante, este último mejora la colaboración significativamente
  • Git es automáticamente instalado con algunos programas. Algunos de ellos ampliamente utilizados por economistas (e.g., Python-Anaconda, o command line tools de MacOs, Matlab 2020b)
  • Git no es lo mismo que GitHub! GitHub es una "red social" que utiliza la arquitectura Git como base. Otros ejemplos son GitLab y BitBucket

CÓMO SE USA GIT?

CÓMO SE USA GIT?

  • Git se usa a través de la linea de comando (shell, console, terminal, etc.) e interfaces gráficas de usuario (GUI)

    Linea de comando

  • El uso básico de Git en la linea de comando requiere conocer entre 5 y 10 comandos.

CÓMO SE APRENDE GIT?

  • La mejor forma de aprender Git es usándolo!!
  • La mayor parte de la documentación es a través de la línea de comando.
  • Esto no quiere decir que el usuario lo use a través de ella, pero le da uniformidad al proceso (Las GUI pueden tener formas diversas de implementar Git y aprenderlas es difícil si no se entiende el objetivo. Es como querer aprender MS excel sin saber matemáticas)
  • Con el tiempo, el uso y la terminología Git se vuelve natural

Colaboración con Git

  • Anteriormente mencionamos que git no requiere servidores.
  • En teoría, podríamos compartir un repositorio utilizando un disco duro en común
  • También se pueden crear permisos de lectura y escritura (ver documentación de git init)
  • No obstante, un servidor git permite crear un espacio común y seguro!
  • Git ocupa los siguientes protocolos de comunicación:
    • Local
    • HTTP y HTTPS
    • SSH
    • GIT

Colaboración con Git

  • Para colaborar con git sólo nos hace falta aprender los siguentes comandos
    • git remote
    • git clone
    • git pull (git fetch + git commit)
    • git push

Sin embargo hay un componente humano que hace falta

Plataformas de colaboración (RR.SS.)

  • Tener la información no es lo mismo que entender la información
  • Las "redes sociales" en torno a Git permiten dar fluidez a la comunicación y generar estructuras de colaboración y dominio de código.
    • Tipos de colaboradores (e.g. dueño, desarrollador, observador)
    • Archivos Readme y Wiki
    • Issues
    • Forks
    • Pull/Merge requests
  • Las mas utilizadas son GitHub, GitLab y BitBucket.

Comentarios finales

  • Git es un secretario que lleva registro completo de nuestro trabajo.
  • Es cada dia mas popular... incluso entre economistas. (replicabilidad y colaboración)
  • Git no es mágico. Requiere que los equipos definan estrategias para crear ramas y resolver conflictos.
  • Las GUI's ayudan a humanizar git. Hacen la organización del cambio manejable.
  • Las RR.SS. son herraminetas de colaboración y comunicación muy poderosas. Patio Inglés digital + información no sincrónica

Comentarios finales

  • El uso del ecosistema Git permite ampliar el set de herramientas comunicacionales internas.
  • Al simplificar la gestión del cambio permite acceder a nuevas tecnologías.
  • No todas esas tecnologías son "machine learning" y "big data".
    • Aplicaciones web
    • Dashboards
    • Sitios estáticos
